💬 Messenger

Asymmetric KEM — Per-User Encapsulation

Each participant receives a different cryptographic blob; neither stores the raw key

Alice Server Bob ←── Register ──────────────► kyber.keygen() → (pk_A, sk_A) ←── Register ──────────────► kyber.keygen() → (pk_B, sk_B) Alice selects Bob to chat POST /conversations ────────► S = HKDF(random ‖ random ‖ qrng) ct_A = KEM.encap_with_key(pk_A, S) ct_B = KEM.encap_with_key(pk_B, S) DB: conversation(ct_A, ct_B) ← { key: ct_A } ─────────── ──── WS push: { key: ct_B } ──► Each side decapsulates independently to recover S Messages: AES-256-GCM(msg, S) — raw key never leaves server memory

🛡️ Security Hardening

Critical Fixes Applied

Post-build security audit — all critical issues resolved

🔑

Asymmetric KEM Fixed

Previously both users shared the same raw key. Now each user receives a unique KEM-encapsulated blob — the raw session key is never stored or transmitted.

⏰

JWT Expiry Enforcement

Client now decodes token expiry on load and schedules an auto-logout timer. Expired tokens are discarded immediately on page load.

🚦

Rate Limiting

SlowAPI middleware on FastAPI enforces 5 login attempts/min and 10 registrations/min per IP — prevents brute-force and credential stuffing.

🧹

Production Code Cleanup

Removed all window.__last_* debug globals, dead handleIncomingMessage stub, and console noise from the messenger.
